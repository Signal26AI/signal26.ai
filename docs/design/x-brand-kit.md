## 🔗 Link Strategy

### Analysis

**Linktree/Bio.link Pros:**
- Can include multiple links (all 3 products, GitHub, docs)
- Easy to update without changing bio
- Analytics on click-through rates
- Common pattern, users understand it

**Linktree Cons:**
- Extra friction (additional click)
- Less direct attribution
- Feels slightly less "technical" (more influencer-coded)
- Not necessary if we only have 3 products in MVP stage

**Direct Product Link Pros:**
- Zero friction, one less click
- Better for focused conversion
- More professional for B2D tools
- Can rotate based on launch cycles

**Direct Product Link Cons:**
- Can only promote one thing at a time
- Need to change bio when launching new products
- Other products get less visibility

### Recommendation: **Direct Product Link → signal26.ai**

**Reasoning:**
1. **We're a brand, not just products** — the main site should be the hub
2. **Single domain = stronger brand recall** — signal26.ai can list all products
3. **More professional for dev tools** — matches the "technical but not corporate" vibe
4. **Less friction** — devs exploring Signal26 see all offerings on one page
5. **Future-proof** — as we add more products, we don't clutter a link tree

**Link text in bio:** "→ signal26.ai" or just "signal26.ai" at the end

**Implementation:**
- Main site (signal26.ai) should have clear CTAs for each product
- Product pages can be signal26.ai/shippolicy, etc.
- Keep it fast-loading and mobile-optimized
- Consider a "coming from X?" landing variant that speaks to iOS devs directly

**Alternative for launch week:** If doing a focused launch of ONE product (e.g., ShipPolicy), can temporarily link directly to that product page with signal26.ai/shippolicy, then revert to main domain after launch period.

## 🎯 Brand Voice Guidelines

### Tone Principles
1. **Sharp, not snarky** — Confident without being condescending
2. **Technical, not gatekeepy** — Use dev jargon but explain when needed
3. **Trustworthy, not corporate** — Professional without press release language
4. **Helpful, not salesy** — Solve problems first, sell second

### Voice Examples

**❌ Corporate:** "We're excited to announce the launch of our innovative solution..."  
**✅ Signal26:** "We built ShipPolicy because App Store rejections suck."

**❌ Too casual:** "lmao app store rejected me again bruh"  
**✅ Signal26:** "Three hours debugging why App Store Review flagged NSUserDefaults. Turns out: missing privacy manifest entry. Again."

**❌ Generic dev tips:** "Always test your app before submitting to the App Store!"  
**✅ Signal26:** "Pro tip: `NSUserDefaults`, `NSFileManager`, and `UserDefaults` all require privacy manifest entries now. Apple's static analysis WILL catch these."

### Tweet Formula Templates

**Product update:**
"[Feature] is live. [What it does in one sentence]. [Why it matters to indie devs]."

**Dev tip:**
"[Specific problem we just solved]. Here's how to avoid it: [Tactical advice]."

**Build in public:**
"[Metric or milestone]. [What we learned]. [What's next]."

**Humor/Meme:**
"[Relatable iOS dev pain point]. [Our product] → [Solved state]."

## Notes & Rationale

### Design Decisions
- **Dark color scheme:** Aligns with dev tool conventions (VSCode, Cursor, Raycast) and reduces eye strain
- **Electric blue accent:** Vibrant enough to pop in tiny avatar, associated with tech/digital signals
- **Geometric/minimal logo:** Works at all sizes, memorable, avoids literal interpretations (no clipart radios or lighthouses)
- **Monospace typography vibe:** Signals "developer tool" without being heavy-handed

### Copy Decisions
- **"Tools for iOS devs who ship":** Direct audience targeting (iOS devs), action-oriented (ship), implies speed/efficiency
- **Product names with bullets:** Visual pattern recognition, easy to scan, shows portfolio of tools
- **No buzzwords:** Avoided "innovative," "cutting-edge," "revolutionary" — devs hate marketing speak
- **Specificity over generality:** "Privacy compliance, linting, icons" > "developer productivity tools"

### Link Strategy Decision
- **signal26.ai over link tree:** Builds brand equity in single domain, more professional, less friction
- **Can rotate for launches:** If doing Product Hunt launch, temporarily point to specific product page
- **Future-proofs expansion:** As we add more products, main site scales better than link tree

### Voice Guidelines Rationale
- **"Sharp, not snarky":** Devs appreciate confidence but hate condescension
- **"Technical, not gatekeepy":** Use jargon to signal tribe, but don't exclude juniors
- **"Trustworthy, not corporate":** B2D tools require trust; corporate voice destroys it
- **"Helpful, not salesy":** 80/20 rule from Scout's research — value first, product second
